# Phase Workflow

ResearchLens is rebuilt one phase at a time. Each implementation chat must stay within a single active phase.

## Required workflow

1. Restate the active phase scope before changing files.
2. List deliverables and explicit out-of-scope items for that phase.
3. Keep changes inside the current phase only. Do not pre-build later business logic.
4. Update relevant docs when repository structure, architecture, or configuration changes.
5. Produce a phase completion report before handoff.

## Handoff rule

Future phases should read prior phase reports before continuing. The report history is the running record of what was created, what changed, and which decisions remain open.
